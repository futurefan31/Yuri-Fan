# 项目协作说明

## 用户信息

- 用户名：禹辰。
- 特点：不喜欢繁琐操作，偏好省事、直接可用的方案；喜欢尝试新技术。
- 协作时应主动完成可安全执行的步骤，减少不必要的确认和手工操作；采用新技术前需确认它确实能改善项目，不为新而新。

## 项目概览

- 项目名称：`NEON NOODLE · 未来亚洲料理`。
- 项目类型：无框架、无构建步骤的静态单页网站。
- 页面定位：虚构的上海未来亚洲料理餐厅宣传页，整体采用雨夜、霓虹、赛博都市风格。
- 页面语言：简体中文为主，辅以英文装饰文案。
- 当前没有 `package.json`、依赖管理、测试框架、后端服务或数据库。
- `.nojekyll` 表明该站点可直接按静态文件方式发布，也适合 GitHub Pages 一类的静态托管。

## 文件结构

- `index.html`：完整页面结构与文案，包括顶部导航、首屏、品牌宣言、招牌菜单、沉浸体验、预约入口、页脚和预约弹窗。
- `styles.css`：全部视觉样式、响应式布局、动画和设计变量。
- `script.js`：全部页面交互，包括预约弹窗、移动端导航、滚动显现和预约表单成功态。
- `assets/neon-noodle-hero.png`：首屏背景图，尺寸为 1774 × 887，内容为霓虹雨夜餐厅内景。
- `README.md`：目前只保存了 Git 用户配置命令，不是完整的项目使用文档；其中的个人联系信息不应复制到其他文件。
- `codex-`、`codex-list`：当前为空目录，不承载功能。

## 页面结构与内容

页面从上到下分为：

1. 顶部导航：品牌标识、关于我们、招牌菜单、沉浸体验和预约按钮。
2. 首屏：上海坐标、主标题、品牌介绍、营业时间和地址，并使用餐厅图片作为全屏背景。
3. 品牌宣言：介绍炭火、发酵和当代技法结合的料理理念。
4. 招牌菜单：三道菜品，分别为炙烤和牛、昆布渍金枪鱼和黑蒜叉烧拉面。
5. 沉浸体验：展示 48 个席位、灯光与音乐氛围，以及模拟的实时环境数据卡片。
6. 预约引导：打开原生 `<dialog>` 预约弹窗。
7. 页脚：地址、营业时间和社交平台占位链接。

注意：首屏主标题目前是“极品美女”，与餐厅主题并不一致，但这是现有明确内容。除非用户提出修改，不要擅自更正文案。

## 视觉与样式约定

- 核心配色集中定义在 `styles.css` 的 `:root`：近黑背景、暖白文字、霓虹粉、青色和橙色。
- 字体为 Google Fonts 提供的 `Noto Sans SC` 和 `DM Mono`；离线或无法访问 Google 时会回退到通用字体。
- 视觉语言包括深色背景、霓虹描边、辉光、扫描线、噪点、玻璃拟态和细边框。
- 布局在 800px 和 460px 两个断点下适配移动端。
- 动效包括滚动显现、音频波形和按钮悬停效果；`prefers-reduced-motion` 已提供减少动态效果的兼容处理。
- 修改样式时优先复用现有 CSS 变量和类名，保持霓虹粉与青色作为主要强调色。

## 当前交互逻辑

- 所有 `.reserve-trigger` 按钮都会打开预约弹窗。
- 点击关闭按钮或弹窗遮罩可关闭弹窗。
- 移动端菜单按钮会切换 `.nav.open`，同时更新 `aria-expanded`。
- `IntersectionObserver` 为 `.reveal` 元素添加 `.visible`，触发滚动显现。
- 日期输入框的最早日期会在浏览器加载时通过 ISO 日期设定；当前实现按 UTC 取日期，东八区凌晨可能与本地日期相差一天。
- 提交预约表单后只在当前页面显示成功提示；数据不会发送、保存或持久化。

## 已知边界与占位内容

- 项目没有真实预约接口，任何涉及预约落库、短信或通知的需求都需要新增后端或第三方服务。
- “查看完整菜单”和页脚社交平台链接目前都指向 `#`，属于占位链接。
- 页面依赖外部 Google Fonts，但其余核心资源均为本地文件。
- 预约成功后，重新打开弹窗不会自动恢复表单；当前实现是一次性的演示流程。
- 没有自动化测试、代码检查或格式化配置，改动后需要手动检查桌面端和移动端页面。

## 本地运行与验证

- 这是静态站点，不需要安装依赖或执行构建。
- 可直接打开 `index.html`，但建议从项目根目录启动任意静态文件服务器，以更接近部署环境。
- 每次修改至少验证：页面可加载、背景图和字体回退正常、锚点导航可用、移动端菜单可开合、预约弹窗可开关、表单必填校验与成功态正常、窄屏无横向溢出。

## 修改原则

- 优先保持纯静态、零依赖结构；只有需求确实需要时才引入框架、构建工具或后端。
- HTML、CSS 和 JavaScript 的职责应继续分离，不要把大段样式或脚本内联进 `index.html`。
- 新增交互时同步考虑键盘操作、ARIA 状态、移动端布局和减少动态效果设置。
- 不要将真实密钥、环境变量、联系方式或预约数据提交到仓库。
- 不要把 `.DS_Store`、依赖目录、构建产物、日志或本地环境文件纳入版本控制；现有 `.gitignore` 已覆盖这些常见文件。
- 修改现有文案、价格、地址、营业时间和品牌设定前，应以用户的明确要求为准。
